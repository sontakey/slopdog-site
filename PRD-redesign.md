# PRD: Slopdog.com Full Visual Redesign

## Overview

Complete visual redesign of slopdog.com using the **frontend-design** (Impeccable Design) skill. The site currently suffers from generic "AI slop" aesthetics: cyan-on-dark, uniform card grids, no motion, thin typography. This redesign makes it look like it was designed by a human with taste, not generated by an AI.

## Design Context (.impeccable.md)

**Target audience**: Music fans interested in AI culture, tech-savvy 18-35, hip-hop listeners, AI industry insiders curious about creative AI applications.

**Use cases**: Discover Slopdog's music, listen to tracks, read behind-the-scenes content, buy merch, understand what a "fully automated AI music artist" is.

**Brand personality/tone**: Confident, irreverent, slightly subversive. Think Kanye's DONDA site meets a high-end streetwear brand. Not "techy" or "hacker" in the cheap green-terminal way. Premium, editorial, with attitude. The robot character is the mascot but the site shouldn't look like a sci-fi movie prop.

## Current Problems (from audit)

1. **AI color palette**: `#00ff41` neon green on `#050a06` black is the single most common AI-generated design choice. It screams "a bot made this."
2. **Uniform card grids**: Every section uses identical cards with the same padding, same borders, same hover states.
3. **No visual rhythm**: Same spacing everywhere. No tension, no breathing room, no surprise.
4. **Thin, low-contrast typography**: Light gray (#eaeaea) body text on near-black. Only one font (Space Grotesk) used identically everywhere.
5. **No motion**: Zero entrance animations, no hover micro-interactions worth noting, completely static.
6. **Terminal/hacker gimmicks**: `EXECUTE_READ.exe`, `ENTER_EMAIL_ADDRESS`, glitch effects, scanlines. These were clever once, now they're AI cliche.
7. **About page**: Wall of MDX in a single bordered card. No visual storytelling.
8. **Missing imagery**: Album cards and merch cards render blank (images exist in `/public/images/` but may have path mismatches).

## Design Direction

**Aesthetic**: Editorial luxury meets streetwear. Think high-end music magazine layout with raw, confident typography. Dark palette but NOT the AI palette.

**Color palette** (replace entirely):
- Background: Deep warm charcoal, NOT pure black. Something like `oklch(12% 0.01 60)` (warm-tinted near-black)
- Primary accent: Shift from neon green to something unexpected. Options: warm amber/gold, hot coral, electric violet. Pick ONE and commit. The robot character's teal LED smile can remain a secondary accent in artwork but shouldn't drive the whole site palette.
- Text: Off-white with warmth, not pure #eaeaea. Tint neutrals toward the accent hue.
- Use the OKLCH color system from the skill's color reference.

**Typography**:
- Replace Space Grotesk. It's fine but forgettable.
- Display font: Something with real character. Consider: **Instrument Sans** (bold, geometric), **Syne** (brutalist personality), **Clash Display** (fashion-forward), or **Cabinet Grotesk** (premium streetwear feel). Pick what fits the editorial luxury direction.
- Body font: Can be the same family at lighter weights, or pair with something highly readable.
- Use a proper modular type scale with fluid sizing via `clamp()`.
- Headlines should be LARGE and confident. Not timid 2xl. Think 6xl-8xl on desktop with tight tracking.

**Layout**:
- Break the grid. Use asymmetric compositions. Full-bleed hero images. Overlapping elements.
- NOT everything in max-w-6xl centered containers. Let content breathe and surprise.
- Use CSS Grid creatively. Vary column spans. Create tension between tight groupings and generous white space.
- Remove the universal card-wrapping pattern. Not everything needs a border and rounded corners.

**Motion**:
- Staggered entrance animations on page load (fade up with delay, not bounce).
- Smooth hover states on interactive elements.
- Use `prefers-reduced-motion` to respect accessibility.
- Exponential easing (ease-out-quart/quint), no bounce or elastic.

**Interactions**:
- Remove terminal/hacker copy (EXECUTE_READ.exe, etc.). Replace with confident, short copy.
- Make the audio player feel premium, not like a terminal widget.
- Merch size selector should feel tactile and responsive.

## Page-by-Page Spec

### Homepage (`src/app/page.tsx`)

**Hero**:
- Full-viewport hero with the latest release's cover art as a large, dominant visual (not constrained in a small square).
- Track title in massive typography overlaid or adjacent.
- Streaming links (Spotify, Apple) as clean icon buttons, not blocky CTAs.
- Audio player redesigned: minimal, sleek, integrated into the hero.
- Kill the "Live Now" badge and `LATEST_DROP:` terminal formatting.

**Discography section**:
- Break from the 4-column identical grid. Feature the newest release larger, with older releases in a different layout (maybe a horizontal scroll, or stacked list with large cover art on hover).
- Album art should be the hero of each card. Minimal text overlay.

**Blog section**:
- Editorial layout. Maybe a featured post with a large image and two smaller ones beside it, rather than 3 identical cards.
- Remove `// LOG_ENTRY` and `> EXECUTE_READ.exe`.

**Newsletter section**:
- Redesign from the current neon-green block. Make it subtle and integrated, not a screaming banner. A simple inline signup in the footer or a minimal section.

### Music Page (`src/app/music/page.tsx`)

- Gallery/editorial layout. Large cover art is the focus.
- Consider a list view option alongside grid (like Bandcamp or a record store).
- Each entry shows: cover art (large), title, date, concept snippet.
- Remove `OPEN` badges.

### Music Detail (`src/app/music/[slug]/page.tsx`)

- Full-width cover art at the top (like a vinyl record sleeve display).
- Lyrics in clean, readable typography. Not inside a bordered card.
- Streaming links as clean buttons.
- Beat license section should be clear but not dominate.

### Blog Page (`src/app/blog/page.tsx`)

- Editorial grid. Featured post larger. Mix of card sizes.
- Clean date formatting (not `// LOG_ENTRY`).
- Tags as subtle pills, not bordered boxes.

### Blog Detail (`src/app/blog/[slug]/page.tsx`)

- Medium-style reading experience. Wide content, beautiful typography.
- Feature image full-bleed at top.
- Minimal chrome around the content.

### Merch Page (`src/app/merch/page.tsx`)

- E-commerce feel. Clean product grid.
- Product images should be visible and large (fix any path issues).
- Size selector and buy button should feel premium.
- Stem pack promo can be a featured banner at top.

### About Page (`src/app/about/page.tsx`)

- Visual storytelling, not a wall of text.
- Break content into sections with visual breaks.
- Pipeline visualization: could be a nice horizontal flow or timeline.
- The Slopdog character SVG could anchor the page visually.

### Header (`src/components/Header.tsx`)

- Clean, minimal. SLOPDOG wordmark (not the `>_` terminal icon).
- Remove glitch effects from nav.
- Sticky header with backdrop blur is fine, but refine the styling.

### Footer (`src/components/Footer.tsx`)

- More substantial. Social links, newsletter signup, navigation, credits.
- Make it feel like the end of a magazine, not an afterthought.

## Technical Constraints

- **Framework**: Next.js 16 App Router + TypeScript + Tailwind CSS 4
- **Content**: MDX-based (don't change content structure or MDX files)
- **Stripe checkout**: Preserve `/api/checkout` route and BuyButton functionality
- **SEO**: Preserve all JSON-LD schemas, metadata, sitemap, robots
- **Images**: All images exist in `/public/images/`. Fix any path mismatches but don't regenerate images.
- **Google Fonts**: Use `next/font/google` for any new fonts (same as current Space Grotesk setup)
- **Analytics**: Keep GA4 and Ahrefs scripts in layout
- **Accessibility**: WCAG AA contrast minimums. `prefers-reduced-motion` support.

## Impeccable Design Checklist

Before marking complete, verify against the frontend-design skill's "AI Slop Test":

- [ ] No cyan-on-dark palette
- [ ] No purple-to-blue gradients
- [ ] No uniform card grids (vary sizes, layouts)
- [ ] No gradient text on headings/metrics
- [ ] No glassmorphism used purely decoratively
- [ ] No bounce/elastic easing
- [ ] No monospace-as-lazy-tech-vibes
- [ ] No cards-inside-cards
- [ ] No identical spacing everywhere
- [ ] No centered-everything layout
- [ ] Typography is distinctive and well-paired
- [ ] Color palette uses tinted neutrals, not pure black/white
- [ ] Motion is purposeful with exponential easing
- [ ] Would a human designer be proud of this?

## Success Criteria

1. The site passes the "AI Slop Test" from the Impeccable Design skill.
2. `npm run build` succeeds with zero errors.
3. All existing functionality preserved (Stripe checkout, MDX rendering, SEO schemas, analytics).
4. All images render correctly (fix any path issues found during audit).
5. WCAG AA contrast on all text.
6. Mobile responsive (test at 390px width).
7. A human designer would believe another human designed this.
